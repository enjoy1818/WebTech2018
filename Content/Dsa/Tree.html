โครงสร้างแบบต้นไม้

โครงสร้างแบบต้นไม้ เป็นโครงสร้างข้อมูลที่ประกอบด้วย 2 ส่วน คือ
1. โหนด (Node) เป็นกลุ่มข้อมูลในโครงสร้างต้นไม้
2. กิ่ง (Branch) เป็นส่วนที่ใช้เชื่อมโหนดต่างๆในโครงสร้างต้นไม้

คำศัพท์ที่เกี่ยวกับโครงสร้างต้นไม้

Root : โหนดแรกสุดของต้นไม้
Parent : โหนดที่อยู่สูงกว่าโหนดที่พิจารณา 1 ระดับ
Child : โหนดที่อยู่ต่ำกว่าโหนดที่พิจารณา 1 ระดับ
Sibling : โหนดที่อยู่ระดับเดียวกันและมีพ่อเดียวกัน
Leaf : โหนดที่ไม่มีลูก
Ancestor : โหนดที่อยู่ในเส้นทางการเดินจากรูทมายังโหนดที่พิจารณา
Descendent : โหนดที่อยู่ในเส้นทางการเดินจากโหนดที่พิจารณา ไปจนหมดต้นไม้
Degree : จ่านวนลูกของโหนดที่พิจารณา
Level : ระยะทางจากรูทมายังโหนดที่พิจารณา
Depth : ความสูงของต้นไม้ (= Level ของโหนดที่อยู่ห่างจากรูทมากที่สุด + 1)
Subtree : ต้นไม้ย่อย

ต้นไม้แบบไบนารี(Binary Trees)

Binary Tree เป็นโครงสร้างต้นไม้ ที่แต่ละโหนดจะมีโหนดลูกได้ไม่เกิน 2 ต้นไม้ย่อย
ถ้าต้นไม้มีทั้งหมด N โหนด : Depth สูงสุดจะเท่ากับ N , Depth น้อยสุดจะเท่ากับ log ฐาน 2 ของ N + 1
ถ้าต้นไม้มีความสูง D : จำนวนโหนดน้อยสุดจะเท่ากับ D , จำนวนโหนดมากสุดจะเท่ากับ D**2 - 1

Complete and Nearly Complete Trees

Complete Trees : มีจำนวนโหนดสูงสุด เมื่อมีความสูง D
Nearly Complete Trees : มีความสูงน้อยสุด เมื่อมีโหนด N โหนด

Binary Tree Traversals

Depth-first traversal : เริ่มท่องจากรูท ไปใน descendent ของลูกตัวแรกจนหมด ค่อยไปท่องในลูกตัวถัดไป
Breadth-first traversal : เริ่มท่องจากรูทแบบแนวกว้าง ไปยังลูกทุกตัวก่อน แล้วค่อยท่องไปที่ระดับลูกของลูกไปเรื่อยๆ

Depth-first Traversals

Depth-first Traversals สามารถทำได้ 3 วิธี คือ
1. Preorder traversal : ท่องไปตามลำดับ root -> left subtree -> right subtree
2. Inorder traversal : ท่องไปตามลำดับ left subtree -> root -> right subtree
3. Postorder traversal : ท่องไปตามลำดับ left subtree -> right subtree -> root
