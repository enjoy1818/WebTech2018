Hashing





คือรูปแบบการเก็บข้อมูลเพื่อต้องการให้การค้นหาข้อมูลมีประสิทธิภาพเกือบเป็น O(1) นั่นคือ สามารถเข้าถึง ข้อมูลได้โดยตรง แต่ว่าการเก็บข้อมูลนั้นมีรูปแบบเฉพาะ แต่ว่าไม่ซับซ้อน หลักการคือข้อมูลที่จะนำเข้า หรือค้นหา จะเรียกว่า คีย์ (key) ช้ Hash Function หาตำแหน่งของคีย์นั้นที่เก็บข้อมูล ซึ่งเรียกว่า ตารางแฮช (Hash Table) แต่ว่าก็ยังมีปัญหาที่เรียก ว่า Collision ปัญหาอาจเกิดการชนกันของข้อมูล นั่นคือ คีย์ไม่เหมือนกันแต่ แฮชแล้วได้ค่าเดียวกันในตารางแฮช เช่น กำหนด Hash Function เป็นการ mod ด้วย 100 และมีคีย์ เป็น 10275 และ 27675 เมื่อผ่านการแฮชแล้วจะได้ 75 เหมือนกัน -> เกิดการชนกัน การหลีกเลี่ยงการชนกัน (Collision) กำหนด Hash Function ให้ดี นำคีย์ไปแฮชอีกครั้ง เรียกว่า Rehashing ซึ่งการ rehashing นั้นก็สามารถทำได้หลายวิธีเช่น


1.Linear Probing เมื่อเกิดการชนกัน จะไล่ไปตำแหน่งถัดไป นั่นก็คือ การทำ Rehashing ด้วยฟังก์ชัน (ตำแหน่งที่แฮชได้ + ค่าคงที่) % ขนาด อาร์เรย์


2.Buckets คีย์ที่ผ่านการแฮชแล้วได้ค่าเหมือนกัน สามารถเก็บอยู่ในตำแหน่ง เดียวกันได้ ซึ่งมีการกำหนดจำนวนของคีย์ที่ตำแหน่งเดียวกัน ชัดเจน


3.Chaining (Linked List Collision Resolution) คีย์ที่ผ่านการแฮชแล้วได้ค่าเหมือนกัน สามารถเก็บอยู่ในตำแหน่ง เดียวกันได้ โดยไม่จำกัดจำนวน โดยเก็บข้อมูลตารางแฮชเป็นแบบลิงค์ลิสต์
