Stacks

สแต็ค

สแต็ค เป็นโครงสร้างข้อมูลแบบ Linear List ประเภทหนึ่ง
โดยที่สแต็คจะเพิ่มหรือลบข้อมูลได้จากด้านบนสุด(top)ของสแต็คเท่านั้น
สแต็คมีโครงสร้างเป็นแบบเข้าทีหลัง ออกก่อน (LIFO : Last In First Out)

การดำเนินการพื้นฐานของสแต็ค

สแต็คมีการดำเนินการพื้นฐานอยู่ 3 อย่าง คือ
1. Push
2. Pop
3. Stack Top

Basic Operation : Push

Push เป็นการเพิ่มข้อมูลเข้าไปที่ส่วน top ของ stack ข้อมูลที่เพิ่มเข้าไป
ก็จะอยู่ตำแหน่ง top ของ stack แทนข้อมูลก่อนหน้าที่ถูกดันลงไป

ปัญหา : ถ้าสแต็คเต็ม การเพิ่มข้อมูลลงไปจะทำให้เกิด Overflow

Basic Operation : Pop

Pop เป็นการลบข้อมูลที่อยู่ที่ตำแหน่ง top ของ stack แล้วคืนค่าข้อมูลนั้น
ทำให้ข้อมูลก่อนหน้าจะอยู่ตำแหน่ง top ของ stack แทน

ปัญหา : ถ้าไม่มีข้อมูลหรือสแต็คว่าง การลบข้อมูลจะทำให้เกิดสถานะ Underflow

Basic Operation : Stack Top

คืนค่าข้อมูล ณ ตำแหน่ง top ของสแต็ค แต่ไม่ได้ลบข้อมูลนั้น

ปัญหา : ถ้าสแต็คว่าง การดึงข้อมูลที่ตำแหน่ง top จะทำให้เกิดสถานะ Underflow
