Queues

คิว

คิว เป็นโครงสร้างข้อมูลแบบ Linear List ประเภทหนึ่ง
โดยที่ คิวจะเพิ่มข้อมูลได้ที่ส่วนท้าย(rear)ของคิว
และจะลบข้อมูลได้ที่ส่วนหัว(front)ของคิวเท่านั้น
คิวมีโครงสร้างเป็นแบบเข้าก่อน ออกก่อน (FIFO : First In First Out)

การดำเนินการพื้นฐานของคิว

คิวมีการดำเนินงานพื้นฐานอยู่ 4 อย่าง คือ
1. Enqueue
2. Dequeue
3. Queue Front
4. Queue Rear

Basic Operation : Enqueue

Enqueue เป็นการเพิ่มข้อมูลเข้าที่ส่วนท้าย(rear)ของคิว
ทำให้ข้อมูลนั้นอยู่ที่ตำแหน่งสุดท้ายของคิว

ปัญหา : ถ้าคิวเต็ม การเพิ่มข้อมูลลงไปจะทำให้เกิดสถานะ Overflow

Basic Operation : Dequeue

Dequeue เป็นการลบข้อมูลออกจากคิวที่ส่วนหัวของคิว
แล้วคืนค่าข้อมูลนั้น ทำให้ข้อมูลถัดไปอยู่ที่ตำแหน่งหัวคิวแทน

ปัญหา : ถ้าคิวว่าง การลบข้อมูลออกจะทำให้เกิดสถานะ Underflow

Basic Operation : Queue Front

Queue Front เป็นการส่งค่าข้อมูลที่ตำแหน่งหัวคิวออกมา โดยที่ไม่ได้ลบข้อมูลนั้น

ปัญหา : ถ้าคิวว่าง การพยายามดึงข้อมูลที่ตำแหน่งหัวคิวจะทำให้เกิดสถานะ Underflow

Basic Operation : Queue Rear

Queue Rear เป็นการส่งค่าข้อมูลที่ตำแหน่งท้ายคิวออกมา โดยที่ไม่ได้ลบข้อมูลนั้น

ปัญหา : ถ้าคิวว่าง การพยายามดึงข้อมูลที่ตำแหน่งท้ายคิวจะทำให้เกิดสถานะ Underflow
